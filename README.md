# Claude / Claude Code 実務導入ハンドブック

このフォルダは、Claude への現場移行に向けた HTML 教材を作るための作業場です。

いきなり HTML を作るのではなく、まず以下の順番で品質を固めます。

1. 公式情報の棚卸し
2. 教材構成の確定
3. 各章の学習ゴールと演習設計
4. HTML 画面構成とコンテンツ部品化
5. HTML 実装
6. ブラウザ確認と修正

## なぜ最初から HTML にしないか

最初から HTML を生成すると、見た目の完成度に意識が寄って、次の問題が起きやすくなります。

- 公式情報の参照漏れ
- CLI、設定、権限、MCP、Hooks などの実務項目の抜け
- 章ごとの粒度のばらつき
- 初学者向け説明と現場運用向け説明の混在
- 後から更新しづらい巨大な 1 ファイル化

そのため、この教材では Markdown の設計書をソースとして作り、内容を固めてから HTML に変換します。

## 想定する完成形

完成物は、社内研修や自己学習で使える 1 つの HTML 教材を想定します。

- 左サイドバーで章移動
- 本文に解説、図解、コマンド例、注意点
- 各章に到達目標とチェックリスト
- CLI コマンドやプロンプト例にコピー操作
- 演習と現場導入テンプレートを付録化

## 更新方針

Claude / Claude Code は更新が速いため、HTML 化の直前に公式ドキュメントを再確認します。特にモデル名、価格、CLI フラグ、管理設定、ベータ機能は変更されやすい前提で扱います。
