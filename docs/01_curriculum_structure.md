# 教材構成案

作成日: 2026-06-13

## 教材の目的

現場で利用する AI が Claude に移行する前提で、Claude の基礎理解から Claude Code の実務運用までを体系的に学べる教材を作る。

単なる機能紹介ではなく、次の状態を目指す。

- Claude と Claude Code の違いを説明できる
- Claude Code CLI を日常開発で使える
- コード調査、修正、テスト、Git 操作を安全に依頼できる
- `CLAUDE.md`、設定、権限、Hooks、MCP、Skills、Subagents の役割を理解できる
- チーム導入時のルール、セキュリティ、レビュー観点を持てる

## 想定読者

主対象はエンジニア。副対象として、PM、テックリード、情シス、開発組織の管理者も読める構成にする。

## 推奨する章立て

### 1. Claude 全体像

学習ゴール:

- Claude が何をする AI か理解する
- Claude.ai、Claude API、Claude Code の違いを整理する
- モデル、コンテキスト、トークン、入力形式の基本を押さえる

扱う内容:

- Claude の概要
- モデル選択の考え方
- テキスト、画像、ファイル、コードの扱い
- API とアプリの違い
- 最新情報が変わりやすい領域の見方

### 2. Claude の基本的な使い方

学習ゴール:

- 曖昧な依頼を、再現性のある依頼に直せる
- 出力形式や制約を指定できる
- 要約、比較、レビュー、調査、文章作成に使える

扱う内容:

- 役割、目的、制約、前提、出力形式
- 良いプロンプトと悪いプロンプト
- 例示の使い方
- 長文を扱う時の分割
- 事実確認が必要な場面

### 3. プロンプト設計

学習ゴール:

- Claude に作業意図を正確に渡せる
- 実務テンプレートを作れる
- プロンプトを改善する観点を持てる

扱う内容:

- 明確な指示
- XML 風タグによる構造化
- Few-shot / multishot
- Chain of thought を直接出させない設計
- チェックリスト化
- 評価しやすい出力形式

### 4. Claude Code 入門

学習ゴール:

- Claude Code が何をするツールか理解する
- Terminal / IDE / Desktop / Web の違いを説明できる
- 最初のコード調査と小さな修正を実行できる

扱う内容:

- Claude Code の概要
- Agentic coding tool としての動き
- ファイル読み取り、編集、コマンド実行
- 初回起動とログイン
- 権限確認の基本

### 5. Claude Code CLI 完全ガイド

学習ゴール:

- CLI を日常開発の中心ツールとして使える
- コマンド、フラグ、対話モード、セッション管理を理解する
- Windows / PowerShell / WSL での注意点を押さえる

扱う内容:

- インストール
- `claude`、`claude "query"`、`claude -p`
- `claude -c`、`claude -r`
- `claude auth`、`claude update`、`claude install`
- `/help`、`/clear`、`/login`、`/doctor`
- ショートカット
- `!` シェルモード
- バックグラウンド実行
- `--tools`、`--worktree`、`--system-prompt`、`--append-system-prompt`
- トラブルシュート

詳細は [CLI章の詳細設計](03_cli_module_design.md) に分離する。

### 6. 実務ワークフロー

学習ゴール:

- Claude Code に任せる作業を適切に切り出せる
- 調査、計画、実装、検証の流れで進められる
- Git とテストを組み合わせて安全に作業できる

扱う内容:

- コードベース調査
- バグ修正
- 機能追加
- テスト作成
- リファクタ
- ドキュメント更新
- PR レビュー
- コミット、ブランチ、コンフリクト対応

### 7. プロジェクト文脈の管理

学習ゴール:

- Claude に毎回同じ説明をしなくてよい状態を作れる
- チーム標準の開発ルールを文脈として管理できる

扱う内容:

- `CLAUDE.md`
- `.claude/` ディレクトリ
- Memory
- カスタムコマンド
- プロジェクト固有の禁止事項
- 大きなコードベースでの文脈管理

### 8. 応用機能

学習ゴール:

- Claude Code の拡張機能を用途別に選べる
- 自動化、並列化、外部ツール連携を設計できる

扱う内容:

- Skills
- Subagents
- Hooks
- MCP
- Plugins
- Agent SDK
- GitHub Actions / CI 連携
- Worktree と並列作業

### 9. セキュリティと組織導入

学習ゴール:

- AI 利用時の権限とリスクを説明できる
- チーム導入時の最低限のルールを作れる
- 管理者向け設定の観点を持てる

扱う内容:

- ファイル編集とコマンド実行の権限
- 機密情報の扱い
- コードレビュー責任
- 外部サービス連携時の注意
- managed settings
- MCP allow / deny
- ログ、監査、コスト管理

### 10. ハンズオン

学習ゴール:

- 実際の開発タスクで Claude Code を使える
- 教材の知識を手順として再現できる

演習案:

- 既存プロジェクトの構造を説明させる
- 小さなバグを修正する
- テストを追加する
- `CLAUDE.md` を作る
- CLI の `-p` を使ってログを要約する
- Hooks でテスト実行を促す
- PR レビューを実施する
- チーム用 AI 利用ルールを作る

### 11. 付録

含めるもの:

- CLI コマンド早見表
- よく使うプロンプト集
- `CLAUDE.md` テンプレート
- セキュリティチェックリスト
- 導入 FAQ
- 公式リンク集

## 教材の粒度

1 章あたりの目安:

- 説明: 5-10 分
- 操作例: 3-5 個
- 注意点: 3-5 個
- ミニ演習: 1-2 個

全体では 90-120 分の研修、または自己学習 2-3 時間を想定する。
