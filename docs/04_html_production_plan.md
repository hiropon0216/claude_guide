# HTML化の制作計画

作成日: 2026-06-13

## 結論

いきなり HTML を作るより、次の 3 段階で進める方が精度が高い。

1. Markdown で教材設計を固める
2. 章ごとの本文、演習、コマンド表を作る
3. HTML / CSS / JS に落とし込む

## 推奨する制作ステップ

### Step 1. 教材の設計を確定する

成果物:

- 章立て
- 各章の学習ゴール
- 参照する公式情報
- 演習一覧
- HTML 画面構成

完了条件:

- CLI、Claude 基礎、プロンプト、Claude Code、応用、セキュリティが抜けていない
- 公式ソースマップがある
- 章ごとの対象読者が明確

### Step 2. 本文原稿を作る

成果物:

- 章ごとの Markdown 原稿
- コマンド表
- プロンプト例
- 注意点
- 演習

完了条件:

- 初学者が読んでも流れがわかる
- 現場エンジニアが使う具体例がある
- 公式情報のリンクが各章に紐づいている

### Step 3. HTML コンポーネントを設計する

必要な UI:

- サイドバー目次
- 章検索
- 進捗チェック
- コードコピー
- コマンド早見表
- 警告、注意、補足ボックス
- 演習カード
- 公式リンク集

### Step 4. HTML を実装する

ファイル案:

```text
index.html
assets/styles.css
assets/app.js
```

実装方針:

- まずは静的 HTML で完結させる
- 外部ビルドツールは使わない
- 社内配布しやすい構成にする
- JavaScript は検索、コピー、進捗保存に限定する

### Step 5. ブラウザで検証する

確認項目:

- PC 幅で読みやすい
- モバイル幅でも目次と本文が崩れない
- コードブロックが横にはみ出しても読める
- コピー操作が動く
- 章検索が動く
- 公式リンクが切れていない

## HTMLの情報設計

### 左サイドバー

- Claude 全体像
- 基本操作
- プロンプト設計
- Claude Code 入門
- CLI 完全ガイド
- 実務ワークフロー
- プロジェクト文脈
- 応用機能
- セキュリティ
- ハンズオン
- 付録

### 本文

各章は次の型で統一する。

```text
章タイトル
この章でできるようになること
なぜ重要か
基礎説明
実務例
よくある失敗
ミニ演習
公式リンク
```

### 右側または章末

- チェックリスト
- 関連コマンド
- 公式リンク
- 次に読む章

## 初回HTMLで入れすぎないもの

初版では以下を深追いしすぎない。

- API の詳細実装
- 価格の細かい比較
- すべての MCP サーバー例
- すべての Hooks イベント詳細
- すべての CLI フラグ

これらは付録または第 2 版で拡張する。

## 初版で必ず入れるもの

- Claude と Claude Code の違い
- CLI の使い方
- `CLAUDE.md`
- 実務ワークフロー
- Git とテスト
- 権限と安全な使い方
- チーム導入ルール
- よく使うプロンプト例

## 次の作業候補

1. この構成で合意する
2. 各章の本文原稿を作る
3. 公式情報を再確認してソースマップを更新する
4. HTML のワイヤーフレームを作る
5. `index.html`、`assets/styles.css`、`assets/app.js` を実装する
